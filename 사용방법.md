# 타이핑에듀(typingedu) 사용 방법

이 폴더에는 **소스코드**와 **빌드된 실행 파일(`dist`)**이 모두 들어 있습니다.
목적에 따라 둘 중 하나를 쓰세요.

---

## 1. 비개발자 — 바로 보고 싶을 때 (Live Server)

빌드는 이미 끝나 있습니다. `dist` 폴더만 띄우면 됩니다.

### 방법 A) VS Code + Live Server 확장
1. VS Code에서 **`dist` 폴더를 엽니다** (전체 프로젝트 말고 `dist`만!)
   - 중요: `dist`를 루트로 열어야 화면이 안 깨집니다.
2. `dist/index.html`을 우클릭 → **Open with Live Server**
3. 브라우저가 열리며 사이트가 실행됩니다.

> ⚠️ 그냥 `index.html`을 더블클릭(file://)하면 데모가 안 열려요. 반드시 Live Server 같은 http 서버로 열어야 합니다.

### 방법 B) 명령어로 간단 서버 (Node.js 있을 때)
```
npx serve dist
```
표시되는 주소(예: http://localhost:3000)를 브라우저에서 엽니다.

---

## 2. 웹사이트에 올릴 때 (배포)

`dist` 폴더 **안의 모든 파일**을 웹 서버의 루트(또는 서브도메인 폴더)에 업로드하면 끝입니다.
서버·DB 필요 없는 순수 정적 사이트입니다.

> 하위 폴더(예: `사이트주소/typingedu/`)에 올릴 거면 경로가 깨집니다.
> 그 경우 `vite.config.ts`에 `base: '/typingedu/'`를 넣고 `npm run build`로 다시 빌드하세요.

---

## 3. 개발자 — 코드를 수정/이어서 작업할 때

필요: **Node.js** (https://nodejs.org)

```
npm install          # 1) 패키지 설치
cp .env.example .env # 2) 환경변수 파일 만들고 값 채우기
npm run dev          # 3) 개발 서버 실행 (자동 새로고침)
npm run build        # 4) 배포용 빌드 → dist 폴더 생성
```

### 환경변수 (.env)
백엔드(Google Apps Script) 연동에 필요합니다. `.env.example`를 복사해 `.env`로 만들고 값을 채우세요.
- `VITE_GAS_URL` : GAS 웹앱 URL
- `VITE_GAS_SECRET` : 통신 시크릿

### 주요 폴더
- `src/` — 화면/컴포넌트 소스 (React + TypeScript)
- `public/apps/` — 데모 교구 앱들
- `google-apps-script/` — 백엔드(폼 제출 처리 등) 코드
- `dist/` — 빌드 결과물 (배포/실행용)
