세션 작업 내용을 `etc/session/` 폴더에 저장합니다.

## 저장 절차

1. 현재 세션에서 수행한 작업을 정리한다:
   - 변경/생성/삭제한 파일 목록
   - 구현한 기능 요약
   - 미완료 작업 및 다음 단계
   - 발견한 이슈나 주의사항
   - 설계 결정 및 그 이유

2. 사용자가 `$ARGUMENTS`로 추가 메모를 전달한 경우, 해당 내용도 포함한다.

3. `etc/session/` 폴더에 마크다운 파일로 저장한다:
   - **파일명 규칙**: `YYYY-MM-DD-<상세한-영문-키워드-나열>.md`
   - **이름이 곧 검색 키**이므로, 핵심 키워드를 최대한 포함한다. 짧은 슬러그보다 길더라도 검색 가능한 이름이 낫다.
   - 포함할 키워드: 대상 시스템, 작업 유형, 변경한 주요 항목, 핵심 기술 등
   - 좋은 예: `2026-05-19-mongodb-to-gas-migration-ec2-nginx-ssl-deploy.md`
   - 나쁜 예: `2026-05-19-deploy.md` (너무 포괄적, 검색 키워드 부족)
   - 하루에 여러 세션이면 `-2`, `-3` 접미사 추가
   - 기존 파일 목록을 확인하여 같은 날짜의 기존 세션과 중복되지 않도록 한다.

4. 저장할 내용의 구조:

```markdown
# 세션 기록 — YYYY-MM-DD #NN

## 작업 요약
(한줄 요약)

## 변경 파일
- `경로/파일명` — 변경 내용 설명

## 구현 내용
(상세 설명)

## 인프라 / 배포
(서버, DNS, SSL 등 인프라 관련 작업이 있으면 기술)

## 미완료 / 다음 단계
- [ ] 항목 1
- [ ] 항목 2

## 메모
(추가 사항, 주의점, 설계 결정 등)
```

5. 저장 완료 후 사용자에게 파일명과 요약을 보고한다.

## 주의사항
- 가능한 상세하게 작성하여 다음 세션에서 컨텍스트를 빠르게 복구할 수 있도록 한다.
- 코드 스니펫은 포함하지 않되, 어떤 파일의 어떤 부분을 변경했는지는 명확히 기술한다.
- 환경변수, 비밀번호, API 키, 비밀키 등 민감 정보는 절대 포함하지 않는다.
- EC2 IP, 도메인, 인증서 만료일 등 인프라 정보는 포함한다.
